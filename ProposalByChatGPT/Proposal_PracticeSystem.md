## 1. 專案概要（Overview）

本專案旨在建立一套線上作答系統，將既題目整理後呈現在網頁介面，提供使用者進行作答、對答案與詳解確認。
系統希望達到簡潔易用、功能明確，並可持續擴充題目的目標。

---

## 2. 開發目的（Goals）

在準備基本情報技術者試験時，考慮到紙本參考書的侷限性，  
例如反覆翻頁，關鍵詞查詢，  
以及容易對題目產生圖像記憶，導致無法確實將知識應用在考試上，  
更重要的是考慮多語言對應以增進學習效率，  
因此企劃開發此自我學習與測驗平台，提供自我學習與測驗平台。  

並且藉由此作為契機透過實作練習前端與後端開發技能的證明。  

將參考書題目系統化、資料化，提升複習效率。  

---

## 3. 系統功能（Features）

---

### MVP（最小可行產品）

- 顯示題目列表。

- 題目詳細內容頁面。

- 作答後顯示正確與否。

- 題目資料以 JSON 管理。

### 進階功能（Optional）

- 題目分類、標籤。

- 搜尋功能。

- 隨機出題／亂數答案。

- 題目完成率統計。

- 使用者作答紀錄（LocalStorage／資料庫）。

- 後端 API：題目管理、提交答案、記錄數據。

---

## 4. 技術架構（Technical Stack）

---

### 前端（Primary）

- React + TypeScript

- Vite or CRA

- React Router

### 後端（Optional）

- Node.js + Express

- API 端點提供題目資料與作答檢查

### 資料儲存

- 開發初期：JSON 檔案

- 進階版本：MySql

## 6. 資料格式（Question Schema）

---
```
{
  "id": 1,
  "type": "single",
  "question": "以下哪個是 IPv4 位址？",
  "options": ["255.255.255.0", "fe80::1", "abcd", "192.168.0.1"],
  "answer": [3],
  "explanation": "IPv4 格式為四段 0–255 的十進位。"
}
```

## 7. 開發流程（Development Plan）

---

### 階段 1：前置作業

- 蒐集題目資料並整理成 JSON 格式。

- 定義題目 schema 與類型。

### 階段 2：前端開發

- 建置基本 React 架構

- 題目列表頁面

- 題目詳細頁面 + 作答 UI

- 判斷答案邏輯

- 顯示正確 / 錯誤結果

### 階段 3：優化

- 加入題目分類

- UI 優化

- LocalStorage 進度紀錄

### 階段 4：後端（選擇性）

- 建立 Express API

- 提供題庫資料

- 接收答案、計算結果

- 儲存作答紀錄

---

## 9. 風險與對策（Risks & Mitigation）

---

| 風險           | 可能影響      | 解決方案                        |
| ------------ | --------- | --------------------------- |
| 題目資料龐大，格式不一致 | 題庫載入失敗、錯誤 | 設計嚴謹的 question schema；寫驗證工具 |
| 不同題型邏輯複雜     | 作答判斷錯誤    | 統一題型 interface，模組化判斷流程      |
| 缺乏 UI/UX     | 使用體驗下降    | 逐步優化，使用既有 UI 套件             |
